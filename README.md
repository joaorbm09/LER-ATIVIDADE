### 7 Tecnicas de levantamento de requisitos  
    
#### Briefing:
- coleta de dados ou informações inicias, realizada com o cliente, que será a base para o desenvolvimento inicial do projeto. Essa coleta ocorre em uma reunião com o cliente, chamada reunião de briefing;
- Manual de idenidade visual (cores, logo, slogan, Fontes...)
    - Concorrentes
    - Prazos 
    - Orçamentos 
    - Identidade da Empresa (Missão, Visão, Valores)
    - Publico alvo
#### Brainstorm/ Brainstorming:
- técnica utilizada para levantar sugestões e ideias, a fim de solucionar problemas no desenvolvimento de produtos e serviços;
    - Uma reunião criativa com  o objetivo de solucionar um ou mais **problemas** 
    - Podem ser estruturados ou não-estruturados
    - Reunião Curta com foco em definido
    - ferramentas 
        - Miro
        - mentimeter
        - oneNote
#### Questionário:
- Lista de perguntas sobre os requisitos do projeto. Diferente do briefing, que faz coleta de dados sobre o produto ou serviço a ser desenvolvido, o design, a empresa et., aqui se faz perguntas sobre a implementação do produto ou serviço no mercado;
- perguntas direcionadas ao cliente, com o objetivo de montar o escopo do projeto 
- formas de fazer o questionario
    - reunião com perguntas diretas (qualitativas)
    - formulario de coleta de dados (quatitativas)
#### Entrevistas:
- Técnica para coletar de dados de usuários de um produto ou serviço para identificação de problemas, a partir de suas observações;
    - inatrumento de coleta de dados com o usuario do projeto
    - entrevistas podem ser formais e informais
    - nessa etapa é necessario um MVP (produto minimo viavel)
#### Etnografia:
- Técnica de observação utilizada para analisar requisitos sociais e culturais das tarefas desempenhadas numa organização;

#### Workshop:
- Reunião composta pela equipe de analistas da empresa contratada para desenvolvimento do produto ou serviço e por representantes do cliente, a fim de especificar as necessidades do projeto
    - Tecnica de levantamento de requisitos que acontecem na apresentação de um MVP, para um grupo seleto de pessoas (stakeholders) com intuito de absorver a reação do publico ao produto/projeto
    - Eventos, feiras, e workshops de produtos especificos
    - "Stakeholders? - pessoas que possuem interesse/publico-alvo, funconarios que estão envolvidos no projeto".

#### Prototipagem:
- Modelo preliminar ou inicial do produto que apresenta alguns funcionalidades que ele terá;
    - Criaçao de um elemento de design para simulação de um projeto, com intuito de testar funiconalidades e elementos graficos antes de começar a escrever código 
    - Os prototipos podem ser de:
        - Baixa fidelidade (rafe): rascunho de um desenho pode ser feito a mão. 
        - Media fidelidade (rough): desenhos no computador para remarcação de espaços e layouts, não precisa de texto nem de cores.
        - Alta fidelidade: prototipo focado nas aplicações de funcionalidades iguais aos do projeto que serão desenvolvidas posteriormente em códigos.
